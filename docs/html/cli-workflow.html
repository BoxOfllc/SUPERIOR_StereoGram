CLI Workflow Guide

DepthForge ships a 9-command Click CLI. All commands read a depth map (or image for AI estimation) and write an output image. Commands accept presets, patterns, safety limits, and batch inputs.

How to Invoke

# Activate venv first (Windows)
.venv\Scripts\Activate.ps1

# All commands use this entry point:
python -m depthforge.cli.main <command> [options]

# Short alias if installed as package:
depthforge <command> [options]

# Built-in help
python -m depthforge.cli.main --help
python -m depthforge.cli.main sirds --help
Depth Map Convention All commands expect depth maps where white = near (close to camera) and black = far (background). This matches the standard MiDaS/ZoeDepth output convention. Use --invert-depth if your depth map is reversed.

sirds — Classic Magic Eye Stereogram

depthforge sirds [OPTIONS] DEPTH_MAP OUTPUT
Generate a random-dot SIRDS from a depth map. The output image encodes 3D depth — cross your eyes or use the "magic eye" parallel-focus technique to see it.
FlagDefaultDescription
--presetmediumUse a named preset: shallow, medium, deep, cinema, print, broadcast
--depth-factor0.4Parallax multiplier −1.0 to 1.0. Higher = more 3D depth.
--max-parallax0.033Max shift as fraction of image width (1/30 = ergonomic limit)
--convergence0.5Depth value at zero parallax (screen plane). 0.5 = mid-scene.
--oversample1Internal render scale (1–4). 2 = 2× resolution, Lanczos downsample.
--seedNoneRandom seed for reproducible output
--bilateralTrueApply bilateral filter to depth map before synthesis
--bilateral-d9Bilateral filter diameter
--dilation0Depth dilation kernel size (fills small holes)
--fallofflinearDepth falloff curve: linear, gamma, s_curve, log, exp
--invert-depthFalseFlip near/far before synthesis
--safe-modeFalseHard-clamp to ergonomic limits
--widthsource widthOutput width in pixels
--heightsource heightOutput height in pixels
 
# Basic SIRDS with default settings
python -m depthforge.cli.main sirds depth.png out_sirds.png

# Cinema preset with more depth
python -m depthforge.cli.main sirds --preset cinema depth.png cinema.png

# Strong depth, reproducible seed, 2× oversampled quality
python -m depthforge.cli.main sirds --depth-factor 0.7 --seed 42 --oversample 2 \
    depth.png high_quality.pngbash

texture — Tiled-Pattern Stereogram

depthforge texture [OPTIONS] DEPTH_MAP OUTPUT
Like SIRDS but uses a tileable image (custom or procedural) as the pattern layer instead of random dots.
FlagDefaultDescription
--pattern-fileNonePath to a custom tileable image (PNG/JPEG)
--pattern-nameNoneUse a built-in pattern from the library (e.g. bubbles, circuit, plasma)
--pattern-typerandom_noiseProcedural generator: random_noise, perlin, plasma, voronoi, geometric_grid, mandelbrot
--tile-size128Pattern tile width/height in pixels
--depth-factor0.4Parallax depth multiplier
--presetmediumNamed preset
 
# Use a custom tileable image as the pattern
python -m depthforge.cli.main texture --pattern-file my_tile.png depth.png out.png

# Use a named built-in pattern
python -m depthforge.cli.main texture --pattern-name circuit depth.png circuit_stereo.png

# Plasma procedural pattern
python -m depthforge.cli.main texture --pattern-type plasma --tile-size 256 \
    depth.png plasma_stereo.pngbash

anaglyph — Red/Cyan 3D

depthforge anaglyph [OPTIONS] SOURCE_IMAGE DEPTH_MAP OUTPUT
Generate an anaglyph image for viewing with red/cyan glasses. Takes both a colour source image and a depth map.
FlagDefaultDescription
--modecolour_anaglyphtrue_anaglyph, grey_anaglyph, colour_anaglyph, half_colour, optimised (Dubois)
--parallax-px20Eye separation offset in pixels
--invert-depthFalseFlip depth convention
 
# Best quality — Dubois optimised mode
python -m depthforge.cli.main anaglyph --mode optimised \
    photo.jpg depth.png anaglyph_out.png

# Grey anaglyph (stronger 3D effect, less colour)
python -m depthforge.cli.main anaglyph --mode grey_anaglyph --parallax-px 30 \
    photo.jpg depth.png grey_anaglyph.pngbash

stereo-pair — Side-by-Side Stereo

depthforge stereo-pair [OPTIONS] SOURCE_IMAGE DEPTH_MAP OUTPUT
Generate a left/right stereo pair for VR headsets, 3D monitors, or cross-eye/wall-eye free fusion.
FlagDefaultDescription
--layoutside_by_sideside_by_side, top_bottom, separate
--eye-balance0.5Left/right eye weight (0.0 = all left, 1.0 = all right)
--background-filledgeFill mode for occlusion gaps: edge, black, mirror
--depth-factor0.4Stereo depth strength
 
# Side-by-side (VR / cross-eye)
python -m depthforge.cli.main stereo-pair --layout side_by_side \
    photo.jpg depth.png sbs.png

# Top-bottom for some 3D displays
python -m depthforge.cli.main stereo-pair --layout top_bottom \
    photo.jpg depth.png tb.pngbash

hidden — Hidden Image Stereogram

depthforge hidden [OPTIONS] DEPTH_MAP OUTPUT
Encode a shape or text invisibly into dot noise. The hidden content only reveals when the viewer cross-eye or parallel-fuses the image.
FlagDefaultDescription
--shapeNoneBuilt-in shape: circle, star, heart, arrow, diamond
--textNoneText string to encode as hidden image
--mask-fileNoneCustom black/white mask image to encode
--fg-depth0.7Depth value for foreground (visible) shape region
--bg-depth0.3Depth value for background region
--edge-softness2.0Gaussian blur radius on mask edges
 
# Hidden star shape
python -m depthforge.cli.main hidden --shape star depth.png hidden_star.png

# Hidden text message
python -m depthforge.cli.main hidden --text "HELLO" --fg-depth 0.8 \
    depth.png hidden_text.png

# Custom mask (white = hidden shape region)
python -m depthforge.cli.main hidden --mask-file logo_mask.png \
    depth.png hidden_logo.pngbash

batch — Batch Processing

depthforge batch [OPTIONS] INPUT_DIR OUTPUT_DIR
Process an entire folder of depth maps with consistent settings. Supports all output modes.
FlagDefaultDescription
--modesirdsOutput mode: sirds, texture, anaglyph, stereo_pair, hidden
--presetmediumNamed preset applied to all files
--formatpngOutput format: png, jpeg, tiff, webp
--workers4Parallel worker threads
--pattern-nameNonePattern library name (for texture mode)
 
# Batch SIRDS — all PNGs in a folder
python -m depthforge.cli.main batch --mode sirds --preset cinema \
    ./depth_maps/ ./output/sirds/

# Batch anaglyph with 8 parallel workers
python -m depthforge.cli.main batch --mode anaglyph --workers 8 \
    ./depth_maps/ ./output/anaglyph/

# Batch texture with circuit pattern, output JPEG
python -m depthforge.cli.main batch --mode texture --pattern-name circuit \
    --format jpeg ./depth_maps/ ./output/bash

analyze — QC & Safety Analysis

depthforge analyze [OPTIONS] DEPTH_MAP
Run comfort analysis, window violation detection, and photosensitive epilepsy (PSE/flash) safety checks. Outputs a JSON report and optional overlay images.
# Print comfort report to terminal
python -m depthforge.cli.main analyze depth.png

# Save QC overlays (heatmap, histogram, window violations)
python -m depthforge.cli.main analyze --save-overlays depth.png

# JSON report to file
python -m depthforge.cli.main analyze --json-out report.json depth.pngbash

presets — List Built-in Presets

python -m depthforge.cli.main presets
# Lists: shallow, medium, deep, cinema, print, broadcast
# Shows depth_factor, max_parallax_fraction, convergence for eachbash

estimate-depth — AI Depth from Photo

depthforge estimate-depth [OPTIONS] IMAGE OUTPUT_DEPTH
Run MiDaS or ZoeDepth on any photograph to generate a depth map. Requires PyTorch (see Installation guide). Model downloads ~400 MB on first use.
FlagDefaultDescription
--modelmidasmidas (faster) or zoedepth (more accurate metric depth)
--deviceautocuda, cpu, or auto (uses CUDA if available)
 
python -m depthforge.cli.main estimate-depth photo.jpg depth_out.png

# ZoeDepth for better metric accuracy
python -m depthforge.cli.main estimate-depth --model zoedepth photo.jpg depth.pngbash

Workflow: Quickstart (5 minutes)

Get a stereogram from any photo in three commands.

1

Generate a depth map from your photo

python -m depthforge.cli.main estimate-depth my_photo.jpg depth.pngbash
2

Check the depth looks good

python -m depthforge.cli.main analyze depth.pngbash

Review the comfort report. Look for SAFE or LOW risk rating.

3

Generate the stereogram

python -m depthforge.cli.main sirds --preset medium depth.png stereogram.pngbash

Workflow: Batch Folder Processing

Process an entire shoot — estimate depth for all photos, then batch synthesize stereograms.

# Step 1: Estimate depth for all photos in a folder (run in a loop)
mkdir -p depths outputs

for f in photos/*.jpg; do
  base=$(basename "$f" .jpg)
  python -m depthforge.cli.main estimate-depth "$f" "depths/${base}_depth.png"
done

# Step 2: Batch synthesize all depth maps
python -m depthforge.cli.main batch --mode sirds --preset cinema \
    --workers 8 depths/ outputs/bash

Workflow: Full AI Depth → Anaglyph Pipeline

# Photo → AI depth → Dubois optimised anaglyph in one pipeline

python -m depthforge.cli.main estimate-depth --model zoedepth \
    portrait.jpg depth_portrait.png

python -m depthforge.cli.main analyze depth_portrait.png

python -m depthforge.cli.main anaglyph \
    --mode optimised \
    --parallax-px 25 \
    portrait.jpg depth_portrait.png anaglyph_portrait.pngbash

Workflow: Video Sequence

For temporally stable stereogram video, use the web server or Python API — the CLI handles per-frame synthesis. Use batch mode with pre-computed depth maps per frame.

# Extract frames and depths (external tool, e.g. ffmpeg + depth model)
# Then batch synthesize all frames
python -m depthforge.cli.main batch \
    --mode sirds \
    --preset broadcast \
    --workers 8 \
    ./frame_depths/ ./frame_stereo/

# Reassemble to video (ffmpeg)
ffmpeg -framerate 24 -i ./frame_stereo/frame_%04d.png \
    -c:v libx264 -crf 18 stereogram_video.mp4bash

Workflow: Print Delivery

For print output, use the print preset (300 DPI, TIFF, wide-gamut) and oversample for maximum quality.

# High-quality print SIRDS — 2× oversampled, print preset
python -m depthforge.cli.main sirds \
    --preset print \
    --oversample 2 \
    --width 3600 --height 2400 \
    --seed 42 \
    depth_hires.tiff print_ready.tiffbash
