Full Reference & Glossary

Complete reference for every CLI command, flag, enumeration, pattern name, and technical term in DepthForge. Use Ctrl+F to search this page.

Shared Depth & Stereo Flags

These flags appear on sirds, texture, and most other synthesis commands:

FlagTypeDefaultDescription
--depth / -dPATHrequiredPath to depth map image (PNG, TIFF, EXR, JPEG). White=near, black=far.
--output / -oPATHrequiredOutput file path. Extension determines format (.png, .jpg, .tiff).
--presetSTRINGNoneApply a named preset. Overrides depth-factor, max-parallax, oversample, dpi, and depth prep settings. Values: shallow medium deep cinema print broadcast
--depth-factorFLOAT0.35Parallax strength multiplier. 0.0=flat, 1.0=maximum depth. Negative = pops toward viewer. Range: −1.0 to 1.0.
--max-parallaxFLOAT0.0333Maximum pixel shift as fraction of image width. 1/30 ≈ 0.033 is the ergonomic limit. Increase to 0.05 for dramatic effect.
--safe-modeFLAGoffHard-clamps depth_factor to ±0.5 and max-parallax to 1/30. Recommended for public content. Reduces PSE risk.
--oversampleINT1Internal render scale. 1=off, 2=2× resolution then Lanczos downsample. Dramatically improves quality; 4× slower at oversample=2.
--seedINTNoneRandom seed for reproducible output. None=different every run. Set any integer for identical results across runs.
--invert-depthFLAGoffFlip near/far before synthesis. Use when your depth map has black=near, white=far (some CG Z-passes).
--bilateral-spaceFLOAT5.0Spatial sigma for bilateral edge-preserving filter on depth map. Higher = more smoothing over distance.
--bilateral-colorFLOAT0.1Color/intensity sigma for bilateral filter. Higher = blends across larger depth jumps (flattens edges).
--dilationINT3Morphological dilation radius in pixels. Expands bright (near) regions to fill depth map holes. Set 0 to disable.
--falloffCHOICElinearDepth remapping curve. See FalloffCurve enum. Values: linear gamma s-curve logarithmic exponential
--near-planeFLOAT0.0Clip depth values below this before synthesis. Range 0.0–1.0. Use to cut out very dark background.
--far-planeFLOAT1.0Clip depth values above this. Use to prevent extreme foreground parallax.
--dpiINT72Output DPI embedded in file metadata. Use 300 for print. Affects pixel dimensions only when used with print presets.
--verbose / -vFLAGoffPrint detailed progress messages during synthesis.

Command: sirds

FalloffCurve Enum

Controls how raw depth values are remapped before synthesis. Applied in --falloff flag and DepthPrepParams.falloff_curve.

ValueCLI stringEffect
FalloffCurve.LINEARlinearNo remapping. Raw depth values used as-is. Good starting point.
FalloffCurve.GAMMAgammaGamma curve (power=2.2). Boosts mid-range depth separation. Good for photos.
FalloffCurve.S_CURVEs-curveSigmoid S-curve. Compresses extremes, expands mid-range. Best for most natural scenes.

GPU Synthesis — depthforge.core.gpu_synthesizer

GPU-accelerated SIRDS synthesis using PyTorch tensor operations. No CUDA compiler required — implemented entirely with torch.scatter_reduce_, torch.gather, and torch.arange. Automatically selects CUDA > MPS (Apple Silicon) > CPU-torch > NumPy fallback.

best_device()

from depthforge.core.gpu_synthesizer import best_device
device = best_device()   # 'cuda' | 'mps' | 'cpu'python

Returns the best available torch device string. Priority: CUDA > MPS > CPU. Returns 'cpu' if torch is not installed.

synthesize_gpu(depth, pattern, params, device='auto')

from depthforge import synthesize_gpu, best_device
import numpy as np

depth   = np.random.rand(1080, 1920).astype(np.float32)  # float32 (H, W), values 0–1
pattern = np.random.randint(0, 255, (128, 128, 4), dtype=np.uint8)  # RGBA tile

result = synthesize_gpu(depth, pattern)           # auto device
result = synthesize_gpu(depth, pattern, device='cuda')    # force CUDA
result = synthesize_gpu(depth, pattern, device='cpu')     # force CPU-torchpython

depthfloat32 (H, W)Depth map, values in [0, 1]. White = near.

devicestr'auto' | 'cuda' | 'mps' | 'cpu'. 'auto' selects best available.

Algorithm — Pointer Doubling

The serial bottleneck in the Thimbleby-Inglis algorithm is the transitive chain walk in the same[] array (each pixel follows a chain to its root). GPU reformulation:

  1. Compute all (x_left, x_right) constraint pairs simultaneously via vectorised shift arithmetic — zero Python loops.
  2. Build same[] with a single scatter_reduce_(amin) call — handles multiple constraints per pixel correctly.
  3. Resolve transitive chains with pointer doubling: ceil(log₂(max_shift + 2)) rounds of torch.gather — typically 6–7 passes.
  4. Assign color indices as root_position % tile_W — one modulo operation.
  5. Gather RGBA pixels from the pattern tile in one batched index lookup.

All H rows are processed simultaneously. Chains are bounded by max_shift (~60–100 px), so the number of pointer-doubling rounds is constant regardless of image size.

Glossary — Stereogram Terms

SIRDS
Single Image Random Dot Stereogram. A flat image containing hidden 3D depth information encoded in a pattern of random dots. Invented by Tyler and Clarke (1990), popularised as "Magic Eye". Viewed by relaxing focus to see through the screen plane (parallel fusion) or crossing the eyes.
SIS
Single Image Stereogram. Broader term covering SIRDS and texture-based stereograms. SIRDS is a subset of SIS where the pattern is random noise.
Parallax
The horizontal offset between corresponding points in the left and right eye views. Positive parallax = object appears behind screen. Negative parallax = object appears in front of screen. Zero parallax = object at the screen plane.
Screen plane / Zero parallax surface
The depth level at which objects have zero horizontal offset — they appear to sit exactly at the physical screen surface. Controlled by the convergence parameter.
Vergence
The inward rotation of the eyes to converge on a nearby point. Excessive vergence demand causes viewer fatigue. DepthForge's ComfortAnalyzer checks vergence against SMPTE ST 2098-1 limits.
Window violation
Occurs when an object with negative parallax (popping toward viewer) is clipped by the image border. Creates an impossible depth cue that breaks fusion. DepthForge detects and reports window violations in the analyze command.
Anaglyph
A stereoscopic image made by encoding left and right views in different colour channels (typically red and cyan). Viewed with matching coloured glasses. DepthForge supports 5 anaglyph algorithms including Dubois-optimised.
Stereo pair
Two separate left and right eye views of a scene, with horizontal offset proportional to depth. Can be viewed side-by-side (cross-eye or wall-eye fusion), through a VR headset, or on a 3D monitor.
Fusion
The brain's process of merging the slightly different left and right eye images into a single 3D percept. Magic Eye / SIRDS images require sustained focus control to achieve fusion — the viewer must hold their eyes at a focal distance different from the screen.
depth_factor
DepthForge's master parallax control. Scales the raw depth map values before synthesis. Higher values produce more apparent 3D depth. Values above ~0.5 can cause discomfort for many viewers.
max_parallax_fraction
A safety ceiling on parallax expressed as a fraction of image width. 1/30 ≈ 0.033 is the widely-cited ergonomic limit for comfortable stereoscopic viewing at typical screen distances.

Glossary — Depth Map Terms

Depth map
A greyscale image where each pixel's brightness encodes distance from the camera. DepthForge convention: white = near (close to camera), black = far (background). This matches MiDaS, ZoeDepth, and most ComfyUI depth model outputs.
Relative depth
Depth map where values represent relative ordering (near/far) rather than metric distances. MiDaS produces relative depth. Suitable for stereogram synthesis.
Metric depth
Depth map where values encode real-world distances (in metres). ZoeDepth produces metric depth. DepthForge normalises metric depth to [0,1] before synthesis.
Bilateral filter
An edge-preserving smoothing filter. Blurs flat regions but preserves depth discontinuities at object boundaries. Applied to depth maps before synthesis to reduce noise without softening object edges. Controlled by --bilateral-space and --bilateral-color.
Dilation
A morphological operation that expands bright (near) regions in the depth map. Used to fill small holes and thin gaps in the near layer. Controlled by --dilation. Set to 0 to disable.
Falloff curve
A transfer function applied to depth values before synthesis. Remaps the linear [0,1] range into a different distribution to control how depth differences translate to parallax. S-curve is best for most natural scenes.
Near plane / Far plane
Clipping limits for depth values before synthesis. --near-plane 0.1 --far-plane 0.9 would discard the darkest 10% and brightest 10% of depth values, useful for removing noisy extremes.

Glossary — Safety Terms

PSE — Photosensitive Epilepsy
A condition where certain visual stimuli (flashing lights, high-contrast patterns) can trigger seizures. DepthForge includes a flash_safety module that implements W3C WCAG 2.3, ITU-R BT.1702-2, and the Harding PSE Test to flag high-risk content. Always use --safe-mode for publicly distributed content.
Michelson contrast
A measure of luminance contrast: (L_max − L_min) / (L_max + L_min). DepthForge monitors this for pattern tiles. Values above 0.5 in rapidly-changing patterns exceed recommended limits for PSE-safe content.
safe_mode
A flag that simultaneously: (1) hard-clamps depth_factor to ±0.5, (2) limits max_parallax_fraction to 1/30, (3) limits pattern contrast. Adds a brief PSE acknowledgement to stderr. Use for broadcast, public displays, and general distribution.
WCAG 2.3 / Success Criterion 2.3.1
Web Content Accessibility Guidelines rule limiting flashing content to no more than 3 flashes per second below the general flash threshold. DepthForge's flash_safety module checks against this.
Harding PSE Test
Industry-standard photosensitive epilepsy test used in broadcast. DepthForge implements a simplified version in depthforge.core.flash_safety.

Glossary — Technical Terms

Thimbleby-Inglis algorithm
The core stereogram synthesis algorithm in DepthForge. For each scanline, builds a "same" constraint array: each depth value determines how far apart two pixels must be to appear at the same depth to the viewer. The algorithm then assigns pattern colours left-to-right while honouring those constraints. Reference: Thimbleby, Inglis & Witten (1994).
Oversample
Internal render scale factor. At oversample=2, the stereogram is synthesised at 2× resolution then downsampled with Lanczos filtering. Produces sharper sub-pixel depth transitions. 4× slower at oversample=2; only use for final quality renders.
Tiled rendering
Splitting a large image into horizontal strips and synthesising each strip independently in parallel threads. Used by depthforge.core.tiled_render for 4K/8K output. Strips must overlap by the maximum parallax amount to avoid seam artifacts.
Temporal smoothing
Blending depth maps across adjacent video frames to prevent flickering in video stereograms. DepthForge supports EMA, windowed, flow_guided, and adaptive strategies in depthforge.core.temporal.
OFX
Open Effects Architecture — a plugin standard for visual effects hosts including After Effects, Premiere Pro, DaVinci Resolve, and Nuke. DepthForge's OFX plugin (depthforge/ofx/plugin.cpp) implements this standard, bridging C++ host calls to the Python synthesis engine.
HAS_CV2 / HAS_SCIPY / HAS_TORCH
Package-level boolean flags in DepthForge indicating which optional packages are available at runtime. All synthesis code checks these flags and falls back to pure NumPy/Pillow implementations when optional packages are absent.
GPU synthesis / gpu_synthesizer
DepthForge's PyTorch-based SIRDS synthesizer (depthforge.core.gpu_synthesizer). Replaces the two serial Python for-loops in the Thimbleby-Inglis algorithm with vectorised tensor operations over all H rows simultaneously. Achieves 50–350× speedup over NumPy depending on hardware. No CUDA compiler required — uses standard PyTorch tensor ops.
Pointer doubling
A parallel algorithm technique that resolves transitive chains in O(log N) passes instead of O(N) serial steps. In GPU synthesis, each pixel's same[] chain may point through multiple hops to reach its root. Pointer doubling applies ceil(log₂(max_shift + 2)) rounds of torch.gather(same, 1, same) — each round doubles the number of hops resolved. Typically 6–7 rounds for standard parameters.
scatter_reduce_ (amin)
PyTorch operation used to build the same[] constraint array. For each valid (x_left, x_right) pair, atomically sets same[row, x_right] = min(same[row, x_right], x_left). The amin reduction correctly handles multiple constraints targeting the same pixel — the leftmost root wins, ensuring correct stereogram colour propagation. Requires PyTorch ≥ 1.12; falls back to plain scatter_ on older versions.
MiDaS / ZoeDepth
AI monocular depth estimation models available in DepthForge. MiDaS v3.1 DPT_Large (~400 MB) produces relative depth suitable for stereograms. ZoeDepth N (~700 MB) produces metric depth in metres, normalised to [0,1] by DepthForge. Both auto-download on first use via torch.hub; cached at ~/.cache/torch/hub/. Accessible via depthforge estimate-depth CLI, the Nuke gizmo's EstimateDepth modes, and depthforge.depth_models.get_depth_estimator().
