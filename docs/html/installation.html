Installation Guide

This guide covers installing DepthForge for standalone Python/CLI use and setting up each integration surface (ComfyUI, Nuke, OFX hosts, web preview).

Platform Developed on Windows 11. All surfaces run on Windows, macOS, and Linux. The OFX plugin requires a CMake build step on non-Windows platforms.

Core Installation

Step 1 — Clone the repository

git clone https://github.com/BoxOfllc/SUPERIOR_StereoGram.git
cd SUPERIOR_StereoGrambash

Step 2 — Create and activate a virtual environment

# Windows (PowerShell)
python -m venv .venv
.venv\Scripts\Activate.ps1PowerShell
# macOS / Linux
python3 -m venv .venv
source .venv/bin/activatebash

Step 3 — Install the package

# Minimal install (NumPy + Pillow only — pure Python, no optional deps)
pip install -e .

# CLI install (adds Click)
pip install -e ".[cli]"

# Full install (all optional dependencies)
pip install -e ".[full]"bash

Optional Package Details

Important — Fallback Architecture Every optional package has a pure NumPy/Pillow fallback. DepthForge works without any optional packages — they only improve quality or enable AI features. The fallbacks are production-tested.

OpenCV

pip install opencv-python

Enables: hardware-accelerated bilateral filtering (depth conditioning), Farneback dense optical flow (video), edge detection for adaptive dot density.

SciPy

pip install scipy

Enables: Gaussian blur via ndimage (smoother depth prep), better inpainting with interpolate, faster distance transforms.

PyTorch + AI Depth Models

# CUDA GPU (recommended for speed)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# CPU only
pip install torch torchvision

Enables: MiDaS v3.1 and ZoeDepth monocular depth estimation — turn any photo into a depth map automatically. Models (~400 MB each) download on first use to ~/.cache/torch/hub/.

OpenEXR

# Windows (pre-built wheel)
pip install openexr

# macOS
brew install openexr
pip install openexr

# Linux
sudo apt install libopenexr-dev
pip install openexr

Enables: reading and writing 32-bit linear EXR depth maps. Falls back to 16-bit TIFF if unavailable.

ComfyUI Integration Setup

ComfyUI uses its own embedded Python You must install DepthForge inside ComfyUI's Python environment, not your system venv.

Option A — Junction Link (Recommended for development)

This symlinks the repo directly into ComfyUI's custom_nodes folder so any code changes are immediately reflected without reinstalling.

# Windows — run as Administrator
$src = "E:\SupStudiosWeb3\ProductionTools\SUPERIOR_StereoGram_v0.5"
$dst = "E:\ComfyUI\ComfyUI_windows_portable_nvidia\ComfyUI\custom_nodes\DepthForge"
New-Item -ItemType Junction -Path $dst -Value $srcPowerShell

Option B — Direct copy

cp -r /path/to/SUPERIOR_StereoGram \
      /path/to/ComfyUI/custom_nodes/DepthForgebash

Install depthforge into ComfyUI's Python

# Windows portable — use the embedded python_embeded pip
$comfyPy = "E:\ComfyUI\ComfyUI_windows_portable_nvidia\python_embeded\python.exe"
& $comfyPy -m pip install numpy pillowPowerShell

After this, launch ComfyUI normally:

E:\ComfyUI\ComfyUI_windows_portable_nvidia\run_nvidia_gpu.bat

You should now see a DepthForge category in the ComfyUI node browser with 12 nodes, and a Swirls category with 10 optical illusion generators.

Verify ComfyUI nodes loaded In ComfyUI's console output, look for: DepthForge: loaded 12 nodes and Swirls: loaded 10 nodes. If you see import errors, ensure numpy/pillow are installed in ComfyUI's embedded Python.

Nuke Integration Setup

DepthForge registers a Python gizmo and toolbar menu into Nuke via menu.py.

Step 1 — Make depthforge importable from Nuke

Add the repo root to Nuke's Python path. Edit (or create) your ~/.nuke/init.py:

import sys
sys.path.insert(0, r"E:\SupStudiosWeb3\ProductionTools\SUPERIOR_StereoGram_v0.5")python — ~/.nuke/init.py

Step 2 — Register the toolbar

Add to your ~/.nuke/menu.py:

import depthforge.nuke.toolbar as df_toolbar
df_toolbar.register()python — ~/.nuke/menu.py

This creates a DepthForge menu in the Nuke toolbar with:

  • Create Stereogram node (Alt+Shift+S)
  • Presets submenu (Shallow / Medium / Deep / Cinema / Print / Broadcast)
  • About DepthForge

Step 3 — Verify

Launch Nuke. From the menu: DepthForge → Create Stereogram. A DepthForge gizmo node should appear in your node graph.

OFX Plugin (After Effects / Premiere / DaVinci)

Build step required The OFX plugin is a C++ binary that must be compiled. Python (depthforge) must also be importable from the host's environment.

Prerequisites

  • CMake ≥ 3.16
  • C++17 compiler (MSVC 2019+ on Windows, Xcode on macOS, GCC on Linux)
  • DepthForge installed in the system Python (not a venv) so the OFX host can find it

Build

cd depthforge/ofx
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Releasebash

Install the bundle

# The build outputs: DepthForge.ofx.bundle/

# Windows — copy to OFX plugin directory
xcopy /E DepthForge.ofx.bundle "C:\Program Files\Common Files\OFX\Plugins\" /I

# macOS
cp -r DepthForge.ofx.bundle /Library/OFX/Plugins/

# Linux
cp -r DepthForge.ofx.bundle /usr/OFX/Plugins/bash

Restart After Effects / Premiere / DaVinci Resolve. The DepthForge Stereogram effect will appear in the Effects panel under DepthForge.

Web Preview Server

pip install -e ".[web]"
python -m depthforge.web.app
# Open http://localhost:5000 in your browserbash

The web server provides a single-page UI with all synthesis controls. Output is returned as a base64 PNG. SSE (server-sent events) streams progress for long renders.

Troubleshooting

IssueFix
ModuleNotFoundError: depthforge in ComfyUI Install numpy/pillow in ComfyUI's embedded Python, not your system Python. Use python_embeded\python.exe -m pip install ...
DepthForge nodes don't appear in ComfyUI browser Check ComfyUI console for import errors. Ensure the junction/symlink points to the correct folder and the folder contains __init__.py.
setuptools.backends.legacy error on Python 3.10 Fixed in pyproject.toml — ensure it uses setuptools.build_meta, not setuptools.backends.legacy.
2 temp-file cleanup errors in test_phase3 / test_phase5 Windows OS quirk — not a code bug. All 396 tests pass; Windows sometimes can't delete temp files before the test completes checking them.
MiDaS/ZoeDepth model download hangs Models are ~400 MB each. First-time download goes to ~/.cache/torch/hub/. Ensure network access and ~1 GB disk space.
OFX plugin can't find Python The OFX bridge calls python via subprocess. Ensure python is on the system PATH and import depthforge succeeds from a plain terminal (not inside a venv).
