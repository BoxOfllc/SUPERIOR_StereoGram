Preset Comparison Samples

All 6 presets rendered against the same synthetic depth map — a central sphere with background hills and foreground bumps. The Perlin greyscale pattern is used consistently so depth differences are visible rather than colour differences.

How to view

To see the 3D effect: relax your focus as if looking through the screen (parallel fusion / "Magic Eye" technique) until the pattern doubles and then locks into a 3D image. Or cross your eyes slightly until the two copies merge. Notice how deeper presets produce more dramatic pop-out depth.

Source Depth Map

Synthetic depth map — 800 × 600

Generated with a central sphere (white = nearest), rolling hills in the background (mid-grey), and three smaller foreground bumps. No AI model required — built from NumPy math.

This is the depth map fed into all 6 presets below. The only thing changing between presets is depth_factor, max_parallax_fraction, and output_dpi.

Reproduce it:

python docs/html/samples/generate_samples.py

Individual Presets

shallow
df=0.20 parallax<=0.025 dpi=72
Subtle depth — very comfortable for all audiences including first-time viewers, children, and those prone to eye strain. Minimal vergence demand. Good for long viewing sessions and public displays.
python -m depthforge.cli.main sirds --preset shallow --depth depth.png --output out.png
medium
df=0.35 parallax<=0.033 dpi=72
The ergonomic default. Balances visible 3D pop with viewer comfort. max_parallax at exactly the 1/30 comfortable-fusion limit. Good for most content and most audiences.
python -m depthforge.cli.main sirds --preset medium --depth depth.png --output out.png
deep
df=0.55 parallax<=0.050 dpi=72
Maximum 3D depth — the sphere and bumps should appear dramatically raised above the background. For experienced stereo viewers and short viewing sessions. Not recommended for public distribution without safe-mode.
python -m depthforge.cli.main sirds --preset deep --depth depth.png --output out.png
cinema
df=0.30 parallax<=0.022 dpi=72
Conservative parallax limits for large-screen projection. At cinema distances the angular vergence demand is higher, so lower parallax fractions are used. DCI / SMPTE ST 2098-1 compliant. Comfortable for 2+ hour screening sessions.
python -m depthforge.cli.main sirds --preset cinema --depth depth.png --output out.png
print
df=0.28 parallax<=0.025 dpi=300
Optimised for printed output. 300 DPI metadata, conservative parallax for arm's-length viewing distance (~60 cm). The key difference is the DPI setting — use with --oversample 2 for best print quality.
python -m depthforge.cli.main sirds --preset print --oversample 2 --depth depth.png --output out.tiff
broadcast
df=0.22 parallax<=0.020 dpi=72
Most conservative preset — designed for TV and streaming where screen size, viewing distance, and audience age/health are unknown. EBU R95 compliant parallax limits. Flattest depth but widest audience compatibility.
python -m depthforge.cli.main sirds --preset broadcast --depth depth.png --output out.png

Regenerate These Samples

The generation script is included in the repo. Run it any time to regenerate all samples:

# Activate venv first (Windows)
.venv\Scripts\Activate.ps1

# Generate all 8 images (depth source + 6 presets + comparison grid)
python docs/html/samples/generate_samples.py

The script creates a synthetic depth map in NumPy, runs all 6 presets through the DepthForge synthesis engine, and saves them to docs/html/samples/. No external depth map or photo needed.
