Nuke Workflow Guide

DepthForge integrates into Nuke as a Python gizmo with a toolbar menu. The gizmo wraps the full DepthForge synthesis engine — all 6 output modes, all presets, full depth conditioning, and QC overlays — in a single node with tabbed controls.

Architecture

The integration consists of two files:

  • gizmo.py — defines the DepthForge Python gizmo class with full knob layout (7 tabs, 50+ parameters)
  • toolbar.py — registers the DepthForge menu in Nuke's toolbar with keyboard shortcuts and presets submenu
Depth Map Convention The DepthForge gizmo expects the depth input where white = near (close to camera) and black = far. This is the standard Nuke depth convention from ZDefocus and most depth-from-defocus workflows. Use the Invert Depth toggle on the gizmo if your depth map is inverted.

Installation

See the full Installation Guide — Nuke section. Quick summary:

import sys

The gizmo has 3 input connectors:

  • depth — depth map (always required)
  • source — colour source image (required for Anaglyph / Stereo Pair modes)
  • pattern — tileable pattern image (required for Texture mode)

Press Alt+Shift+S or use DepthForge → Create Stereogram. Connect the Read node to the depth input.

Open the gizmo properties. In the Stereo Params tab:

  • Preset: Medium (or choose Cinema for stronger effect)
  • depth_factor: 0.4

Click the DepthForge node in the viewer. Connect a Write node to output. Render normally.

  1. Read your beauty/colour render and depth pass separately
  2. Create a DepthForge node (Alt+Shift+S)
  3. Connect beauty render → source input, depth map → depth input
  4. Set Mode: Anaglyph
  5. In the Pattern/Anaglyph tab: anaglyph_mode = optimised (Dubois — best colour fidelity), parallax_px = 20
  6. Preview in the Viewer and adjust parallax_px for desired 3D intensity
Tip — Dubois Optimised The optimised (Dubois) anaglyph mode uses perceptually weighted colour matrices designed for red/cyan glasses. It preserves the most colour while maintaining strong 3D. Use grey_anaglyph for maximum 3D depth at the cost of colour.

Workflow: Stereo Pair for VR / 3D Monitors

  1. Connect beauty render → source, depth → depth
  2. Set Mode: Stereo Pair
  3. In Pattern tab: stereo_layout = side_by_side (or top_bottom for some VR formats)
  4. Set depth_factor to your desired 3D strength
  5. background_fill = edge (best for most content — fills occlusion gaps with edge pixels)
  6. Write to a wide output format (2× width for side_by_side)
Output Resolution Side-by-side stereo pair output is 2× the source width (left and right frames concatenated). Top-bottom output is 2× the source height. Plan your Write node format accordingly.

Python Scripting in Nuke

You can drive DepthForge directly from Nuke's Python Script Editor for automation:

import nuke
import depthforge as df
import numpy as np

# Get the depth from a Nuke node and synthesize in Python
node = nuke.toNode('Read1')
arr = node.execute(0, 0, 1)  # frame 0

# Or use the depthforge API directly
from depthforge.core.synthesizer import StereoParams, synthesize
from depthforge.core.depth_prep import DepthPrepParams, prep_depth
import numpy as np

# Load depth (from file)
from depthforge.io.formats import read_depth
depth = read_depth("depth.exr")  # float32 (H,W) in [0,1]

# Condition and synthesize
params = StereoParams(depth_factor=0.5, seed=42)
prep = DepthPrepParams(bilateral_d=9)
depth_conditioned = prep_depth(depth, prep)
stereo = synthesize(depth_conditioned, params)  # (H,W,3) uint8

# Save result
from PIL import Image
Image.fromarray(stereo).save("stereogram.png")python

Nuke-Specific Tips

Working with EXR depth passes

Nuke renders depth as a 32-bit linear float in the Z channel. Before connecting to DepthForge:

  • Use a Shuffle node to isolate the Z channel and route it to the RGBA input of DepthForge
  • Use a Grade node to normalize Z values to [0,1] — typical CG depth ranges from 0 to far clip distance
  • Enable Invert Depth on the gizmo if your Z pass uses camera-near=0 convention (DepthForge expects white=near)
set cut_paste_input [stack 0]
version 14.0 v6
Shuffle2 {
 fromInput1 {{0 "rgba.red"} {0 "depth.Z"} {0 "depth.Z"} {0 "depth.Z"}}
 name Shuffle_DepthToRGB
}
Grade {
 # Normalize Z range to 0-1 — adjust these for your scene scale
 blackpoint {10 10 10 10}
 whitepoint {1000 1000 1000 1000}
 reverse true
 name Grade_NormalizeDepth
}Nuke script snippet

Animating depth_factor over time

depth_factor can be keyframed to animate the 3D effect strength. In the gizmo, right-click depth_factor → Set Key, then animate:

# In Nuke's Script Editor — animate depth_factor
df_node = nuke.toNode('DepthForge1')
df_node['depth_factor'].setAnimated()
df_node['depth_factor'].setValueAt(0.0, 1)   # Frame 1: flat
df_node['depth_factor'].setValueAt(0.6, 48)  # Frame 48: full depthpython

# Run Nuke in terminal mode to render without GUI

Performance Note The DepthForge gizmo runs Python synthesis on every render. For large-resolution or high-frame-count renders, use the tiled_render path (set in gizmo or via Python API) which distributes work across threads. oversample=2 significantly increases render time but produces sharper output — use for final renders, not previews.
